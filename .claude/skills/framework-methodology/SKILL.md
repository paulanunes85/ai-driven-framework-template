# Framework Methodology Skill

## Overview

This skill provides knowledge of the **5-Phase Delivery Methodology** used across all client engagements. Each phase has defined activities, deliverables, and exit criteria.

## Phases

### Phase 1: Discover

**Objective**: Understand the current state, stakeholders, and objectives.

**Activities**:
{{PHASE_1_ACTIVITIES}}

**Deliverables**:
- Current State Assessment
- Stakeholder Map
- Scope Definition Document

**Exit Criteria**:
- Stakeholders aligned on scope
- Current state documented
- Key constraints identified

---

### Phase 2: Plan

**Objective**: Define the strategy, roadmap, and resource plan.

**Activities**:
{{PHASE_2_ACTIVITIES}}

**Deliverables**:
- Strategic Roadmap
- Resource Plan
- Risk Register
- Timeline with Milestones

**Exit Criteria**:
- Roadmap approved by stakeholders
- Resources allocated
- Risks documented with mitigation plans

---

### Phase 3: Build

**Objective**: Execute the plan and create deliverables.

**Activities**:
{{PHASE_3_ACTIVITIES}}

**Deliverables**:
- {{BUILD_DELIVERABLES}}
- Progress Reports
- Updated Risk Register

**Exit Criteria**:
- Core deliverables completed
- Quality gates passed
- Stakeholder review completed

---

### Phase 4: Validate

**Objective**: Verify quality, accuracy, and stakeholder acceptance.

**Activities**:
{{PHASE_4_ACTIVITIES}}

**Deliverables**:
- Validation Report
- Issue/Defect Log
- Acceptance Criteria Checklist

**Exit Criteria**:
- All acceptance criteria met
- Issues resolved or documented
- Stakeholder sign-off obtained

---

### Phase 5: Deliver

**Objective**: Hand off deliverables and ensure knowledge transfer.

**Activities**:
{{PHASE_5_ACTIVITIES}}

**Deliverables**:
- Final Deliverable Package
- Knowledge Transfer Documentation
- Lessons Learned Report
- Transition Plan

**Exit Criteria**:
- All deliverables handed off
- Knowledge transfer completed
- Project closed

## Usage

When creating engagement plans or documents, reference these phases to ensure completeness. Replace `{{PHASE_ACTIVITIES}}` placeholders with domain-specific activities relevant to the engagement.
