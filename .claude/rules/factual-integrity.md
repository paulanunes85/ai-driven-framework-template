# Factual Integrity Rules

These rules apply to all markdown files in the workspace.

## Citation Requirements

1. Every metric, statistic, KPI, benchmark, or market claim must include a hyperlinked source.
2. Format: `[Source Name](https://url)` — inline for single facts, or collected in a `## References` section.
3. Credible sources only: GitHub, Microsoft, Anthropic, Azure, Gartner, IDC, McKinsey, IBM, Forrester, DORA Metrics, IEEE, ACM, official vendor documentation.
4. When no source exists, mark as `[ESTIMATE]` or `[ASSUMPTION]` with reasoning.
5. Never present estimates, projections, or assumptions as established facts.
6. When writing about market trends, always perform a web search to verify data currency.

## Prohibited Patterns

- Unattributed percentages: "85% of enterprises..." without a source link
- Vague attributions: "studies show...", "research indicates...", "experts say..."
- Fabricated benchmarks: invented ROI figures, made-up cost savings, fictional timelines
- Outdated data presented as current: always check publication date
